This folder is for storing data files and code used in your research.

Recommended structure:
- Raw data files (CSV, Excel, JSON, etc.)
- Processed/cleaned data files
- Analysis scripts (R, Python, MATLAB, etc.)
- Statistical output files
- Any other supplementary materials

Best practices:
1. Use clear, descriptive filenames
2. Include a data dictionary or codebook explaining your variables
3. Document your code with comments
4. Keep raw data separate from processed data
5. Include README files explaining the purpose of each file
6. Consider version control for your code (Git)

Example files you might include:
- analysis.R or analysis.py (main analysis script)
- data_cleaning.R or data_preprocessing.py
- raw_data.csv (original data)
- cleaned_data.csv (processed data)
- results_summary.xlsx
- figures_generation.R (script to create figures)

Note: Large datasets may be better stored in external repositories
(e.g., Zenodo, Figshare, OSF) and referenced in your thesis.
